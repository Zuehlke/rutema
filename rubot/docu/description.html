Rubot Workflow

Rubot is a distributed system consisting of a master server that coordinates one or more build slaves.
The server coordinates and evaluates build requests and handles all reporting and archiving tasks while the slaves execute the builds.

The Rubot Master

Two concepts are vital in RubotMaster: the build request and the change.

A change is a set of file changes that may of may not require a build. It contains information about the version control system, the revisions of the files, author information, commit comments etc.

A build request is what the rubot master creates when a build is to take place. It is always directed at a single build slave and contains everything that is needed to execute and describe the build (except the result of course): The steps that comprise the build process, the version of the sources to use and the change that triggered the build. Note that a build request does not necessarily require a change.
